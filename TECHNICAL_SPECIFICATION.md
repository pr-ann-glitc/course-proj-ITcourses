## 1. Описание проекта

### 1.1 Название системы
Система управления учебными IT-курсами

### 1.2 Назначение
Система предназначена для автоматизации работы университета, предоставляющего учебные курсы. позволяет регистрировать студентов, 
создавать и управлять курсами, а также обрабатывать оплату подписки для доступа к курсам (подписка оплачивается один раз и не имеет срока действия).

---

## 2. Функциональные требования

### 2.1 Основные функции системы

### Функция 1: добавление курса
*Описание:* добавление нового курса в базу данных. необходимо ввести название, длительность, день и время проведения, 
максимальное количество учеников, выбрать преподавателя,
по желанию добавить описание к курсу.
автоматически курсу присваивается ID. после сохранения курс отображается в списке.

*Выход:* курс добавлен в базу данных, ему присвоен уникальный номер.


### Функция 2: добавление преподавателя
*Описание:* добавление нового преподавателя в базу данных. необходимо ввести имя, фамилию, образование (профиль обучения), почта.
автоматически преподавателю присваивается ID. после сохранения преподаватель отображается в списке.

*Выход:* преподаватель добавлен в базу данных, ему присвоен уникальный идентификатор.


### Функция 3: регистрация студента
*Описание:* регистрация студента в системе. вводит фио, телефон, email. 
автоматически студенту присваивается ID.

*Выход:* студент добавлен в базу данных, ему присвоен уникальный идентификатор.


### Функция 4: формирование расписания
*Описание:* формирование расписания на неделю с курсами, на которые записан студент, а также получение информации о пересечениях в расписании. 
можно посмотреть расписание на неделю с курсами, на которые записан выбранный студент, при пересечении времени курсов будет выведено сообщение.

*Выход:* расписание на неделю


### Функция 5: оплата подписки
*Описание:* оплата подписки для получения доступа к курсам. пользователь вводит данные карты, 
система проверяет корректность и при успешной оплате курсы доступны для прохождения.

*Выход:* платеж обработан, курсы доступны

## 3. Структура базы данных

```
### Таблица 1: Courses (Курсы)

Описание: Хранит информацию о курсах

Поля:
- ID (int, PK) - уникальный номер курса
- Name (string) - название курса
- Duration (int) - длительность курса (в месяцах)
- Day (string) - день проведения
- Time (string) - время проведения
- ID_Teacher (int, FK) - преподаватель (ID преподавателя)
- Max (int) - максимальное количество учеников
- Description (strint) - описание курса (nullable)

Связи:
- связь один-ко-многим с таблицей StudentsAndCourses (один студент может быть на нескольких курсах)
- связь один-к-одному с таблицей StudentsPaymentDetails (у одного студента одна карта)
```

```
### Таблица 2: Teachers (Преподаватели)

Описание: Хранит информацию о преподавателях

Поля:
- ID (int, PK) - уникальный идентификатор преподавателя
- FirstName (string) - имя преподавателя
- Surname (string) - фамилия преподавателя
- Education (string) - образование
- Email (string) - почта преподавателя

Связи:
- связь один-ко-многим с таблицей Courses (один преподаватель может вести несколько курсов)
```

```
### Таблица 3: Students (Студенты)

Описание: Хранит личную информацию о студентах 

Поля:
- ID (int, PK) - уникальный идентификатор студента
- Surname (string) - фамилия ученика
- FirstName (string) - имя ученика
- LastName (string) - отчество ученика
- PhoneNumber (string) - номер телефона ученика
- Email (string) - почта ученика

Связи:
- связь один-ко-многим с таблицей StudentsAndCourses (один студент может быть на нескольких курсах)
- связь один-к-одному с таблицей StudentsPaymentDetails (у одного студента одни данные для оплаты)
```

```
### Таблица 4: StudentsAndCourses (Студенты_на_курсах)

Описание: связывает студентов и курсы, хранит информацию о студентах и курсах, на которые они записаны

Поля:
- ID (int, PK) - уникальный идентификатор записи
- ID_Student (int, FK) - уникальный идентификатор студента
- ID_Course (int, FK) - уникальный номер курса
```

```
### Таблица 5: StudentsPaymentDetails (Данные об оплате)

Описание: хранит данные карты студента и состояние подписки

Поля:
- ID (int, PK) - уникальный идентификатор записи
- ID_Student (int, FK) - уникальный идентификатор студента
- CardNumber (string) - номер карты
- ExpirationDate (date) - срок действия карты
- CVV (int) - CVV карты
- SubscriptionStatus (enum) - статус подписки
  - 'Yes' - подписка оплачена
  - 'No' - подписка не оплачена
```
  
## 4. Основные формы приложения

### Форма 1: MainMenu (Главное меню)

**Назначение:** форма меню

**Действия:**
- кнопка `Курсы` - открывает ListCoursesWindow для управления курсами
- кнопка `Преподаватели` - открывает ListTeachersWindow для управления преподавателями
- кнопка `Студенты`- открывает ListStudentsWindow для управления студентами
  
---

### Форма 2: ListCoursesWindow (форма списка)

**Назначение:** форма для управления курсами

**Элементы управления:**
- `DataGrid` - отображение списка курсов из базы данных
- `TextBox (поиск)` - поле для поиска курсов по названию

**Действия:**
- кнопка `Добавить курс` - открывает AddOrEditCourseWindow для создания нового курса
- кнопка `Редактировать курс` - открывает AddOrEditCourseWindow для редактирования выбранного курса
- кнопка `Удалить курс` - удаляет выбранный курс после подтверждения
- кнопка `Добавить учеников на курс` - открывает ListStudentsForAddToCourseWindow для добавления учеников на курс

---

### Форма 3: AddOrEditCourseWindow (форма редактирования)

**Назначение:** форма для создания нового курса или редактирования выбранного

**Элементы управления:**
- TextBox `CourseNameTextBox`- название курса
- ComboBox `TeacherComboBox` - преподаватель
- TextBox `DurationTextBox` - длительность курса
- ComboBox `DayComboBox` - день проведения курса
- TextBox `TimeTextBox` - время проведения курса
- TextBox `MaxTextBox` - максимальное количество учеников на курсе
- TextBox `DescriptionTextBox` - описание курса

**Действия:**
- кнопка `Сохранить`:
  - Если режим создания: добавляет новый курс в БД
  - Если режим редактирования: обновляет существующий курс
  - Закрывает окно с DialogResult = true
  - Возврат на ListCoursesWindow
- кнопка `Отмена`:
  - Закрывает окно без сохранения (DialogResult = false)
  - Возврат на ListCoursesWindow

---

### Форма 8: ListStudentsForAddToCourseWindow (форма списка)

**Назначение:** форма для отображения списка студентов для добавления их на курс

**Элементы управления:**
- DataGrid `AllStudents` - отображение списка студентов из базы данных

---

### Форма 9: ViewStudentWindow (форма просмотра)

**Назначение:** форма для просмотра информации о студенте

**Действия:**
- кнопка `Оплатить подписку`:
  - если подписка не оплачена, то открывается PaymentWindow для оплаты
  - если оплачена, то уведомление об этом
- кнопка `Сформировать расписание` - открывает форму ScheduleWindow для просмотра расписания для выбранного студента
---

### Форма 10: PaymentWindow (форма оплаты)

**Назначение:** форма для оплаты подписки

**Элементы управления:**
- TextBox `StudentSurnameTextBox`- фамилия студента (автоматически заполнена)
- TextBox `StudentNameTextBox`- имя студента (автоматически заполнена)
- TextBox `StudentPatronymicTextBox` - отчество студента (автоматически заполнена)
- TextBox `CardNumber` - номер карты студента
- TextBox `ExpirationDate` - срок действия карты студента
- TextBox `CVV` - CVV карты студента

**Действия:**
- кнопка `Оплатить` - обработка оплаты:
  - успешная оплата - закрытие формы (DialogResult = true)
  - ошибка оплаты - уведомление об ошибке
- кнопка `Отмена` - закрытие формы (DialogResult = false) и возврат на форму ViewStudentWindow

---

### Форма 11: ScheduleWindow

**Назначение:** форма для просмотра расписания

**Элементы управления:**
- `DataGrid` - отображение дней недели и времени, на пересечениях записи о курсах, проходящих в это время
- TextBox `Intersection` - выводит сообщение о пересечениях в расписании

**Действия:**
- кнопка `Ок` - закрытие формы (DialogResult = true) и возврат на форму ViewStudentWindow
